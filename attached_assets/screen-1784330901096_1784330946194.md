Bonne lecture du code. Voici la situation réelle et le plan complet avant de toucher quoi que ce soit.

Diagnostic

Le Watchtower Flutter app a été conçu comme une app local-first :

Couche	Ce qu'il y a	Problème web
DB	Isar (FFI natif)	❌ pas de web
Serveur local	shelf sur localhost:8080	❌ pas de web
Extensions	Exécution JS en isolate	❌ pas de web
Media	media_kit (FFI)	❌ pas de web
Auth	local_auth (biométrie)	❌ pas de web
Web stubs	Faux packages vides	💀 tout est mocké
Réseau	rhttp (binaire natif)	❌ pas de web

Résultat : la version web compile mais est quasiment non-fonctionnelle — elle tourne sur des stubs vides. Le SDK (watchtower_client) lui parle directement au serveur Node.js distant et remplace tout ça d'un coup.

Plan
Étape 1 — Purge des packages natifs

Supprimer de pubspec.yaml tout ce qui ne tourne pas sur web :

isar + isar_flutter_libs → remplacé par shared_preferences (config) + état mémoire Riverpod
shelf + shelf_router → supprimé (le serveur, c'est le Node.js distant)
media_kit + media_kit_video → remplacé par video_player (supporte web)
rhttp → remplacé par http (déjà dans le SDK)
local_auth, window_manager, bitsdojo_window → supprimés
Supprimer web_stubs/ entier
Étape 2 — Nouveau layer de données (SDK)

Remplacer tous les providers Riverpod qui lisent Isar / appellent le serveur local par des providers qui utilisent WatchtowerClient :

ServerConfigNotifier   → lit/écrit SharedPreferences (URL + clé API)
WatchtowerClientProvider → crée WatchtowerClient(url, apiKey) depuis config
SourcesProvider        → client.sources.list()
PopularProvider(id)    → client.sources.popular(id, page)
LatestProvider(id)     → client.sources.latest(id, page)
SearchProvider(id, q)  → client.sources.search(id, q)
DetailProvider(id,url) → client.sources.detail(id, url)
VideosProvider(id,url) → client.sources.videos(id, url)
Étape 3 — Écrans à garder / supprimer
Écran	Action	Raison
Sources browser	✅ Garder + adapter	Core de l'app
Popular / Latest / Search	✅ Garder + adapter	Core
Reader manga / novel	✅ Garder	Fonctionne web
Video player	✅ Garder (swapper media_kit → video_player)	
Remote setup (ConnectServer)	✅ Garder + simplifier	Devient le seul "setup"
Library / Downloads	❌ Supprimer	Nécessite Isar / stockage local
Settings avancés (tunnel, API key locale)	❌ Supprimer	Pour le serveur local uniquement
Settings basiques (thème, langue)	✅ Garder	
Étape 4 — Settings distant

Un seul écran de config :

URL du serveur
Clé API (optionnel)
Bouton Tester + Sauvegarder
Résultat final

Une web app Flutter pure UI — aucun stub, aucun mock, aucune DB locale, aucun serveur embarqué. Elle pointe vers le serveur Node.js déployé, récupère les sources et le contenu via le SDK.

Je lance tout ça ? Ce sera un diff conséquent sur le Watchtower Flutter app — suppression de ~60% du code existant (toute la plomberie locale), réécriture des providers, adaptation des screens. Je te conseille de confirmer avant que je commence.
