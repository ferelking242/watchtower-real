### 3. Architecture multi-UI — pattern retenu

**Problème :** Comment faire coexister plusieurs UIs (TikTok, Netflix, YouTube…) développées dans des repos séparés ?

**Options analysées :** Melos monorepo, git dep pubspec, CI rsync, git submodules, Flutter flavors.

**Décision finale : git dep pubspec.yaml**
- Chaque UI = package Flutter importé via URL git
- `watchtower/pubspec.yaml` ajoute la git dep quand l'UI est prête à être fusionnée
- Flutter pub résout tout automatiquement
- Zero copie de fichiers, zero patch d'imports

**Rejet de Melos :** Force à fusionner tous les repos en un monorepo immédiatement — trop lourd.

---

### 4. Architecture SDK — clarification fondamentale

**Problème initial :** J'avais proposé d'importer un SDK dans watchtower lui-même.

**Correction de l'utilisateur :** Watchtower PRODUIT l'API, il ne la consomme pas. Il n'importe aucun SDK.

**Décision finale :**
```
watchtower → produit l'API → openapi.yaml (spec formelle)
SDK Dart / SDK JS / SDK Python → consomment l'API
Reel, futurs UIs, web, Colab → utilisent les SDKs
```

La décision d'implémenter les SDKs est prise mais pas encore codée.

### 6. Keystore APK permanent

**Problème :** Le workflow build-apk.yml générait un nouveau keystore RSA à chaque build → signature différente → les mises à jour APK cassaient.

**Solution :**
1. Génération d'un keystore permanent via `openssl pkcs12` (keytool absent sur Replit)
2. Encodé en base64 → stocké comme secret GitHub `KEYSTORE_BASE64`
3. Workflow restaure le keystore depuis le secret à chaque build
4. Fallback avec warning si secret absent

**Secrets configurés dans `ferelking242/watchtower-real` :**
- `KEYSTORE_BASE64` ✅
- `KEY_PASSWORD` = `reelwatchtower` ✅
- `STORE_PASSWORD` = `reelwatchtower` ✅

**Règle absolue :** Ne jamais régénérer ce keystore.

---

### 7. Preloading vidéo dans Reel

**Problème :** Chaque page du feed créait son propre Player media_kit et le détruisait au scroll → latence visible.

**Solution :** Pool de Players dans `FeedScreen` :
- Fenêtre [i-1, i, i+1] toujours vivants
- Page active : `player.play()`
- Adjacentes : `player.open(url, play: false)` → buffering en avance
- Hors fenêtre : `player.dispose()`
