## Travail accompli

### 1. SDK — ajout de `getRaw()` et export de `WatchtowerHttpClient`

**Repo :** `ferelking242/watchtower-sdk-dart`  
**Commit :** `c211df4`

**Problème posé :** Les endpoints typés du SDK (`sources.popular()` → `ItemsPage`) ne parsent que les clés `mangas` et `items`. Mais `feed_provider._extractItems()` cherche 8 clés différentes (`list`, `mangas`, `items`, `data`, `results`, `videos`, `posts`, `content`). Passer par les endpoints typés ferait perdre des items pour certaines sources.

**Décision :** Exposer `WatchtowerClient.getRaw(path, {queryParams})` qui effectue un GET avec retry+backoff et auth header, mais retourne le JSON brut non-typé. L'adaptateur utilise `getRaw` pour popular/latest/videos/detail et garde les endpoints typés pour ping/sources.

**Ce qui a été ajouté dans `lib/src/client.dart` :**
```dart
Future<Map<String, dynamic>> getRaw(String path, {Map<String, String?> queryParams}) => _http.get(path, queryParams: queryParams);
```

`WatchtowerHttpClient` est maintenant aussi exporté dans le barrel public.

### 2. watchtower-real — migration `RemoteApiClient` → adaptateur SDK

**Fichiers modifiés :**
- `app/watchtower-real/pubspec.yaml` — ajout git dep `watchtower_client`
- `app/watchtower-real/lib/remote/remote_client.dart` — réécrit comme adaptateur

**Principe de l'adaptateur :**
- Conserve exactement l'interface publique de l'ancien `RemoteApiClient` (mêmes signatures de méthodes)
- `feed_provider.dart` et `remote_config_provider.dart` ne changent pas du tout
- Internalement : `WatchtowerClient` (SDK) gère HTTP, retry, auth

**Mapping méthodes :**

| Méthode ancienne | Délégation SDK | Pourquoi |
|---|---|---|
| `ping()` | `_client.ping()` | Endpoint typé suffisant |
| `sources()` | `_client.sources.list()` + `s.toJson()` | Modèle typé, reconverti en Map |
| `popular(id, page)` | `_client.getRaw('/api/sources/$id/popular')` | Conserve toutes les clés de réponse |
| `latest(id, page)` | `_client.getRaw('/api/sources/$id/latest')` | Idem |
| `search(id, q, page)` | `_client.getRaw('/api/sources/$id/search')` | Idem |
| `detail(id, url)` | `_client.getRaw('/api/sources/$id/detail')` | Idem |
| `videos(id, url)` | `_client.getRaw('/api/sources/$id/videos')` | Idem |

**Nouveau : `dispose()`** — délègue à `_client.close()`. À appeler quand le client n'est plus utilisé (ex: `remoteClientProvider` avec `ref.onDispose`).

## Règles à retenir pour les futurs agents

### SDK getRaw vs endpoints typés
- **Endpoints typés** (`sources.list()`, `ping()`) : utiliser quand les modèles suffisent
- **getRaw** : utiliser quand la réponse peut contenir des clés variables (popular, latest, search, videos, detail) — les sources mangayomi retournent des formats différents selon le type
- **Ne jamais** aller chercher les items uniquement dans `mangas`/`items` — certaines sources retournent `list`, `results`, `posts`, etc.
