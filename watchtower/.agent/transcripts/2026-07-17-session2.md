## Décisions techniques

### Swagger UI sans dépendance npm
**Problème :** npm absent dans l'environnement Replit, `node_modules` inexistant dans le clone.  
**Solution :** Servir le YAML brut via `fs.readFileSync`, pointer Swagger UI (CDN unpkg) directement sur `/docs/openapi.yaml`. Swagger UI 5 parse le YAML nativement côté browser.  
**Règle :** Ne pas ajouter `js-yaml` en dépendance — c'est inutile avec cette approche.

### SDK Dart sans openapi-generator
**Décision :** SDK écrit manuellement (pas généré depuis openapi.yaml).  
**Pourquoi :** L'openapi-generator Dart produit du code verbeux avec des dépendances lourdes (`dio`, `built_value`…). Le SDK manuel est léger (seule dépendance : `http`), typé proprement, et plus facile à maintenir.  
**Règle :** Si la spec change, mettre à jour le SDK à la main — il ne doit pas être regénéré automatiquement.

### Coordination avec l'agent Reel
**Règle :** Cet agent ne touche jamais à `watchtower-real`. Le remplacement de `RemoteApiClient` dans Reel par le SDK Dart est planifié APRÈS stabilisation des features Reel (pagination, settings screen).
