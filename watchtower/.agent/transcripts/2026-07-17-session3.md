### 2. Pagination infinie — `feed_provider.dart`

**Fichier :** `lib/features/feed/providers/feed_provider.dart`

Refactoring complet de `FeedNotifier` pour supporter la pagination :

**Nouveau state de pagination (champs privés) :**
```dart
int    _page              = 1;
bool   _hasMore           = true;
bool   _isLoadingMore     = false;
String _resolvedSourceId  = '';
String _resolvedSourceName = '';
RemoteApiClient? _client;
```

**`_loadPage(client, sourceId, {required int page})`** — extrait de `_loadFromServer`, prend un `page` en paramètre. Passe `page` à `_fetchItems()` → `client.popular(sourceId, page: page)` et `client.latest(sourceId, page: page)`. Résout la source une seule fois (via `_resolveSource()`), met en cache `_resolvedSourceId` et `_resolvedSourceName`.

**`loadMore()`** — méthode publique :
- Guard : `!_hasMore || _isLoadingMore || _resolvedSourceId.isEmpty`
- Incrémente `_page`, appelle `_loadPage(client, _resolvedSourceId, page: _page)`
- Appende les nouveaux items : `state = AsyncData([...current, ...newItems])`
- Rollback `_page--` en cas d'erreur pour permettre de retenter
- Met à jour `loadingMoreProvider` (bool) pour l'indicateur visuel

**Nouveau provider :** `loadingMoreProvider` (`NotifierProvider<_LoadingMoreNotifier, bool>`)

**Détection "plus de pages" :** si `rawItems.length < maxItemsPerPage` (8), on pose `_hasMore = false`.

### 3. Déclencheur pagination — `feed_screen.dart`

**Fichier :** `lib/features/feed/feed_screen.dart`

Dans `onPageChanged` du `PageView.builder` :

if (i >= items.length - 3) {

Charge la page suivante quand l'utilisateur est à 3 items de la fin.

**Spinner visuel :** `_LoadingMoreIndicator` (`ConsumerWidget`) positionné à `bottom: 60` (au-dessus de la bottom nav). S'affiche uniquement quand `loadingMoreProvider == true`.
