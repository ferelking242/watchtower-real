### 2. Fix CI APK (`build-apk.yml`)

**Root cause :** Le workflow générait un keystore `watchtower-debug.keystore` mais `build.gradle` cherche `reel.keystore` (via `KEYSTORE_PATH` env non défini → fallback).

**Fix :**
- Décode `KEYSTORE_BASE64` → `reel.keystore` dans `android/app/`
- Si `KEYSTORE_BASE64` absent → génère un keystore de secours avec keytool (même alias)
- Exporte `KEYSTORE_PATH` vers `$GITHUB_ENV`
- Build `--release` au lieu de `--profile`
- Artifact renommé `reel-arm64-v8a.apk`

**Résultat :** ✅ APK build success dès le premier push fixé.

### 3. Fix CI IPA (`build-ipa.yml`)

**Root cause :** `flutter create --platforms=ios .` génère un projet Xcode avec :
- `ProvisioningStyle = Automatic`
- `DEVELOPMENT_TEAM` vide mais CODE_SIGN_REQUIRED implicitement YES
→ Xcode 16 échoue à la phase de signing même avec `--no-codesign`

Log du failure :
```
Running Xcode build...     47.5s   ← build OK
Failed to build iOS app    ← signing check post-build
Building a deployable iOS app requires a selected Development Team
```

**Fix en deux étapes après `flutter create --platforms=ios .` :**

1. **Patch `project.pbxproj`** (sed) :
   ```bash
   sed -i '' \
     -e 's/DEVELOPMENT_TEAM = [^;]*;/DEVELOPMENT_TEAM = "";/g' \
     -e 's/CODE_SIGN_IDENTITY = "iPhone Developer";/CODE_SIGN_IDENTITY = "";/g' \
     -e 's/ProvisioningStyle = Automatic;/ProvisioningStyle = Manual;/g' \
     ios/Runner.xcodeproj/project.pbxproj
   ```

2. **Patch `Podfile`** (Ruby) :
   Insère dans chaque `target.build_configurations` :
   ```ruby
   config.build_settings['CODE_SIGNING_ALLOWED']  = 'NO'
   config.build_settings['CODE_SIGNING_REQUIRED'] = 'NO'
   config.build_settings['DEVELOPMENT_TEAM']      = ''
   ```

**Résultat :** En attente CI (push 14cd181).
